# T02 — Single source of economic truth

> Economic state must have a single source of truth

## Why this matters

A single canonical ledger. Replicas must converge; replays must be deterministic. Divergence across replicas is the duplication-by-desync class.

## Telemetry

**Event:** `balance_snapshot`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `account` | string | yes | value |
| `balance` | number | yes | numeric value |
| `asset_id` | string | yes | identifier |
| `ledger_seq` | integer | yes | numeric value |
| `ts` | string | yes | ISO-8601 timestamp |

**Example payload:**

```json
{
  "event": "balance_snapshot",
  "account": "<account>",
  "balance": 0.87,
  "asset_id": "<asset_id>",
  "ledger_seq": 42,
  "ts": "2026-05-27T12:34:56Z"
}
```

## Detection signal

```
Balance divergence across replicas OR replay produces different state
```

## Threats this catches

- State desync
- Replica drift
- Inconsistent balance

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Balance snapshots emit per account with ledger_seq, and replica divergence triggers alert. |
| 🟡 **Latent** | Balances are tracked but snapshots are not emitted as events. |
| 🔵 **Collapsed** | Covered by T01 — the ledger itself is the single source of truth and no separate snapshot is needed. |
| ⚪ **Inactive (N/A)** | No persistent balances to reconcile — see T01. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
