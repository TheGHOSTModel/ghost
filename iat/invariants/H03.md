# H03 — Minors protected from exploitation

> Minors must be protected from manipulation or exploitation

## Why this matters

Cross-age-bracket interactions on restricted action types must be evaluated and gated. The signal anchors grooming-pattern and age-gap targeting detection.

## Telemetry

**Event:** `age_gated_interaction`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `actor_player_id` | string | yes | identifier |
| `target_player_id` | string | yes | identifier |
| `target_age_bracket` | string | yes | value |
| `action` | string | yes | value |
| `ts` | string | yes | ISO-8601 timestamp |

**Example payload:**

```json
{
  "event": "age_gated_interaction",
  "actor_player_id": "<actor_player_id>",
  "target_player_id": "<target_player_id>",
  "target_age_bracket": "<target_age_bracket>",
  "action": "<action>",
  "ts": "2026-05-27T12:34:56Z"
}
```

## Detection signal

```
Adult actor targeting minor with restricted action type
```

## Threats this catches

- Grooming pattern
- Age-gap targeting
- Unsolicited contact

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Age brackets are tracked on accounts, and cross-bracket interactions on restricted action types emit a structured event. |
| 🟡 **Latent** | Age is collected but bracket-aware interaction telemetry is not yet emitted. |
| 🔵 **Collapsed** | Covered by H05 (threat signal) — grooming-pattern classification produces the same downstream record. |
| ⚪ **Inactive (N/A)** | The platform is gated to adults at signup with attested verification, so cross-bracket interactions cannot occur. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
