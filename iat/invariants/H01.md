# H01 — Expression must not harm

> User expression must not cause harm to others

## Why this matters

Every expressive act (text, voice transcript, custom name) must be classified before delivery. Missing classifier output for an emitted message is as bad as a high-score one.

## Telemetry

**Event:** `expression_event`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source_player_id` | string | yes | identifier |
| `recipient_scope` | string | yes | value |
| `content_ref` | string | yes | value |
| `classifier_score` | number | yes | numeric value |
| `ts` | string | yes | ISO-8601 timestamp |

**Example payload:**

```json
{
  "event": "expression_event",
  "source_player_id": "<source_player_id>",
  "recipient_scope": "<recipient_scope>",
  "content_ref": "<content_ref>",
  "classifier_score": 0.87,
  "ts": "2026-05-27T12:34:56Z"
}
```

## Detection signal

```
classifier_score crosses threshold OR no classifier output emitted
```

## Threats this catches

- Targeted harassment
- Slur in messaging
- Coordinated brigading

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Every user-generated expression (text, voice transcript, name) is classified before delivery and the score is emitted with the event. |
| 🟡 **Latent** | Classification runs but the score is not emitted with the expression event, or classification is sampled rather than universal. |
| 🔵 **Collapsed** | Folded into H05 (threat signal) — only threat-class classifications produce telemetry; lower-severity classifications stay internal. |
| ⚪ **Inactive (N/A)** | No user-to-user expression — the application has no chat, comments, or user-visible text fields. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
