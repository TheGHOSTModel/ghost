# O06 — Traffic anomalies observable

> Unexpected traffic patterns must be observable

## Why this matters

Per-endpoint traffic baselines, with deviation alerting. Anomalies catch stealth scraping and coordinated probing that volumetric DoS alarms miss.

## Telemetry

**Event:** `traffic_pattern_sample`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `endpoint` | string | yes | value |
| `baseline_qps` | number | yes | numeric value |
| `observed_qps` | number | yes | numeric value |
| `anomaly_score` | number | yes | numeric value |

**Example payload:**

```json
{
  "event": "traffic_pattern_sample",
  "endpoint": "<endpoint>",
  "baseline_qps": 0.87,
  "observed_qps": 0.87,
  "anomaly_score": 0.87
}
```

## Detection signal

```
anomaly_score above threshold OR observed_qps deviating >3σ
```

## Threats this catches

- Stealth scraping
- Targeted scan
- Coordinated probing

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Per-endpoint baselines are computed and observed QPS is compared against them with anomaly scoring. |
| 🟡 **Latent** | Traffic is logged but anomaly detection is not yet running. |
| 🔵 **Collapsed** | Covered by O03 — rate limit telemetry already captures unusual traffic patterns. |
| ⚪ **Inactive (N/A)** | No endpoint traffic to baseline — the application has no inbound request surface. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
