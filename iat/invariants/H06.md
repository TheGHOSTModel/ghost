# H06 — Identity representations controlled

> Harmful identity representations must be controlled

## Why this matters

Identity fields (handle, display name, avatar) are content too. Changes must pass a validator; absence of a validator decision is silent acceptance of slurs and impersonation handles.

## Telemetry

**Event:** `identity_field_change`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `player_id` | string | yes | identifier |
| `field` | string | yes | value |
| `old_value` | string | yes | value |
| `new_value` | string | yes | value |
| `validator_decision` | string | yes | decision/outcome enum |

**Example payload:**

```json
{
  "event": "identity_field_change",
  "player_id": "<player_id>",
  "field": "<field>",
  "old_value": "<old_value>",
  "new_value": "<new_value>",
  "validator_decision": "allow"
}
```

## Detection signal

```
validator_decision == 'flagged' OR validator absent for field type
```

## Threats this catches

- Slur in username
- Impersonation handle
- Hate-symbol avatar

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Identity field changes are validated and the decision is emitted; rejected changes are logged with reason. |
| 🟡 **Latent** | Validation runs but decisions are not emitted as queryable events. |
| 🔵 **Collapsed** | Covered by H01 — identity fields flow through the same expression classifier and event. |
| ⚪ **Inactive (N/A)** | Users have no editable identity fields — handles are system-assigned and not user-controlled. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
