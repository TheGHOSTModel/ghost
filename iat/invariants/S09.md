# S09 — Community actions auditable

> Community actions must be auditable

## Why this matters

Every community or moderation action must log actor, action, subjects, and context. Audit gaps here are the most common source of policy-drift incidents.

## Telemetry

**Event:** `community_action_log`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `actor_id` | string | yes | identifier |
| `action` | string | yes | value |
| `subjects` | array | yes | list of identifiers |
| `context_ref` | string | yes | value |
| `ts` | string | yes | ISO-8601 timestamp |

**Example payload:**

```json
{
  "event": "community_action_log",
  "actor_id": "<actor_id>",
  "action": "<action>",
  "subjects": [
    "<id_1>",
    "<id_2>"
  ],
  "context_ref": "<context_ref>",
  "ts": "2026-05-27T12:34:56Z"
}
```

## Detection signal

```
Action without actor or context reference
```

## Threats this catches

- Untracked mod action
- Phantom community decision
- Audit gap

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Every community or moderation action emits actor, action, subjects, and context reference. |
| 🟡 **Latent** | Actions happen but emitted events are missing context reference or actor. |
| 🔵 **Collapsed** | Covered by H10 — enforcement decisions already carry the required audit fields. |
| ⚪ **Inactive (N/A)** | No community or moderation actions exist in the application. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
