# T06 — Market manipulation detectable

> Market manipulation must be detectable

## Why this matters

Market microstructure patterns (wash, spoof, layering) must be scored. Threshold-crossing scores on these classes are the canonical market-manipulation signals.

## Telemetry

**Event:** `market_pattern_signal`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `instrument` | string | yes | value |
| `pattern_class` | string | yes | value |
| `score` | number | yes | numeric value |
| `actor_set` | string | yes | value |

**Example payload:**

```json
{
  "event": "market_pattern_signal",
  "instrument": "<instrument>",
  "pattern_class": "<pattern_class>",
  "score": 0.87,
  "actor_set": "<actor_set>"
}
```

## Detection signal

```
score > threshold for pattern_class in {'wash','spoof','layering'}
```

## Threats this catches

- Wash trading
- Spoofing
- Layering
- Pump and dump

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Market microstructure pattern classifiers run on order flow and threshold-crossing scores alert. |
| 🟡 **Latent** | Order flow is captured but pattern classifiers are not yet running. |
| 🔵 **Collapsed** | Covered by T07 — collusion-pattern signals overlap and catch the same threats. |
| ⚪ **Inactive (N/A)** | No market or order-matching surface — the application has no two-sided trading. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
