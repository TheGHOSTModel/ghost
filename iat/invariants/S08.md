# S08 — Harassment vs normal distinguishable

> Harassment patterns must be distinguishable from normal activity

## Why this matters

Interaction-pair signals (source to target patterns) distinguish targeted harassment and stalking from normal one-off interactions. Pair-scoped, windowed.

## Telemetry

**Event:** `interaction_pair_signal`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source_id` | string | yes | identifier |
| `target_id` | string | yes | identifier |
| `pattern_class` | string | yes | value |
| `score` | number | yes | numeric value |
| `window` | integer | yes | numeric value |

**Example payload:**

```json
{
  "event": "interaction_pair_signal",
  "source_id": "<source_id>",
  "target_id": "<target_id>",
  "pattern_class": "<pattern_class>",
  "score": 0.87,
  "window": 42
}
```

## Detection signal

```
score > threshold for pattern_class == 'harassment'
```

## Threats this catches

- Targeted harassment
- Stalking pattern
- Pile-on

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Pair-scoped interaction signals are sampled and scored against a harassment pattern class with windowing. |
| 🟡 **Latent** | Interaction data is collected but pair-pattern classification is not yet running. |
| 🔵 **Collapsed** | Covered by H02 — per-user behavior scoring catches the same patterns at a different aggregation. |
| ⚪ **Inactive (N/A)** | No persistent user-to-user interaction — interactions are anonymized or ephemeral. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
