# T05 — Economic actions attributable

> Economic actions must be attributable

## Why this matters

Every economic action carries actor_id and full provenance chain back to a triggering event. Anonymous economic moves are unauditable enrichment.

## Telemetry

**Event:** `economic_action_log`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `actor_id` | string | yes | identifier |
| `action` | string | yes | value |
| `subjects` | array | yes | list of identifiers |
| `provenance_chain` | string | yes | value |

**Example payload:**

```json
{
  "event": "economic_action_log",
  "actor_id": "<actor_id>",
  "action": "<action>",
  "subjects": [
    "<id_1>",
    "<id_2>"
  ],
  "provenance_chain": "<provenance_chain>"
}
```

## Detection signal

```
Economic action without actor_id or provenance chain
```

## Threats this catches

- Anonymous economic move
- Untracked enrichment
- Provenance gap

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Every economic action emits actor_id and a provenance chain back to a triggering event. |
| 🟡 **Latent** | Economic actions are logged but provenance is not captured. |
| 🔵 **Collapsed** | Covered by T08 — high-value actions already require attribution telemetry. |
| ⚪ **Inactive (N/A)** | No economic actions exist in the application. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
