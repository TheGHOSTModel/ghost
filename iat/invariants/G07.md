# G07 — RNG unpredictable, unbiased

> Randomized mechanics must not be predictable or manipulable

## Why this matters

Randomness must be server-seeded, unpredictable, and statistically unbiased. Predictable seed sources or skewed outputs enable reroll abuse and drop manipulation.

## Telemetry

**Event:** `rng_draw`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `draw_id` | string | yes | identifier |
| `seed_source` | string | yes | value |
| `output` | string | yes | value |
| `consumer_action_id` | string | yes | identifier |

**Example payload:**

```json
{
  "event": "rng_draw",
  "draw_id": "<draw_id>",
  "seed_source": "<seed_source>",
  "output": "<output>",
  "consumer_action_id": "<consumer_action_id>"
}
```

## Detection signal

```
seed_source predictable OR output distribution skewed
```

## Threats this catches

- RNG seed prediction
- Reroll abuse
- Drop-rate manipulation

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Authoritative RNG draws are logged with seed source, output, and consumer action; distribution and seed predictability are monitored. |
| 🟡 **Latent** | RNG outputs are logged but seed source or distribution analysis is missing. |
| 🔵 **Collapsed** | Covered by a vendor-supplied audited RNG service that emits its own attestation telemetry consumed elsewhere. |
| ⚪ **Inactive (N/A)** | No randomized mechanics — the application is fully deterministic. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
