# H04 — Explicit content gated

> Sexual or graphic content must be appropriately gated

## Why this matters

Content classification plus viewer age bracket plus gate decision must be co-emitted at render time. Any one missing breaks the auditability of explicit-content exposure.

## Telemetry

**Event:** `content_render`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `content_id` | string | yes | identifier |
| `classifier_tag` | string | yes | value |
| `viewer_age_bracket` | string | yes | value |
| `gate_decision` | string | yes | decision/outcome enum |

**Example payload:**

```json
{
  "event": "content_render",
  "content_id": "<content_id>",
  "classifier_tag": "<classifier_tag>",
  "viewer_age_bracket": "<viewer_age_bracket>",
  "gate_decision": "allow"
}
```

## Detection signal

```
explicit tag rendered to under-age viewer OR no gate evaluated
```

## Threats this catches

- Unfiltered explicit content
- Age-gate bypass
- Image leakage

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Every content render emits the classification tag, the viewer age bracket, and the gate decision. |
| 🟡 **Latent** | Classification and gating exist but the three fields are not co-emitted on render. |
| 🔵 **Collapsed** | Covered by H06 (identity field validation) — content rendering goes through the same validator and decision channel. |
| ⚪ **Inactive (N/A)** | The platform hosts no user-generated or system-provided content that requires age-based gating. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
