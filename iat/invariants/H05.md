# H05 — Threats trigger intervention

> Threats or intimidation must trigger intervention

## Why this matters

High-severity threat classifications must produce a downstream intervention within SLA. A class label without an intervention_id is a system that detected and did nothing.

## Telemetry

**Event:** `threat_signal`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `source_player_id` | string | yes | identifier |
| `classifier_class` | string | yes | value |
| `severity` | number | yes | numeric value |
| `intervention_id` | string | yes | identifier |

**Example payload:**

```json
{
  "event": "threat_signal",
  "source_player_id": "<source_player_id>",
  "classifier_class": "<classifier_class>",
  "severity": 0.87,
  "intervention_id": "<intervention_id>"
}
```

## Detection signal

```
severity >= high without intervention_id recorded within SLA
```

## Threats this catches

- Direct threat
- Doxxing
- Coordinated intimidation

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Threat classifications produce structured events with class, severity, and a linked intervention_id within SLA. |
| 🟡 **Latent** | Threat classification runs but interventions are not yet wired or not yet emitted with the same event. |
| 🔵 **Collapsed** | Folded into H10 (enforcement) — high-severity classifications immediately become enforcement decisions emitted there. |
| ⚪ **Inactive (N/A)** | No expression channel where threats could be issued — no chat, comments, voice, or open-text fields. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
