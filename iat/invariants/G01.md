# G01 — Server authority over outcomes

> The server must be the final authority over action outcomes

## Why this matters

The server holds canonical truth for every action outcome. Clients propose, the server disposes. Without this, any client claim — score, position, kill, vote, bid — can be forged and replayed.

## Telemetry

**Event:** `server_validated_action`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `player_id` | string | yes | identifier |
| `action` | string | yes | value |
| `server_state_hash` | string | yes | value |
| `validation_outcome` | string | yes | decision/outcome enum |
| `ts` | string | yes | ISO-8601 timestamp |

**Example payload:**

```json
{
  "event": "server_validated_action",
  "player_id": "<player_id>",
  "action": "<action>",
  "server_state_hash": "<server_state_hash>",
  "validation_outcome": "allow",
  "ts": "2026-05-27T12:34:56Z"
}
```

## Detection signal

```
validation_outcome != 'pass' OR client_claim != server_truth
```

## Threats this catches

- Outcome spoofing
- Win-claim forgery
- Score injection

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | A server-side authority validates every action submitted by clients, and the validation result is emitted to a queryable log. |
| 🟡 **Latent** | Server-side validation exists but is not emitted as queryable telemetry, or the event is defined but not yet emitted in production. |
| 🔵 **Collapsed** | Validation outcomes are covered by another invariant — e.g. G09 (client claim reconciliation) already emits a per-action server decision. |
| ⚪ **Inactive (N/A)** | The application has no notion of authoritative outcomes that need server adjudication — for example, a fully local single-user tool with no shared state. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
