# T08 — High-value operations logged

> High-value operations must be logged

## Why this matters

Actions classified as high-value (large transfers, top-tier asset moves) must reference an audit channel. Silence on the audit channel for a high-value class is bypassed scrutiny.

## Telemetry

**Event:** `high_value_action`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `actor_id` | string | yes | identifier |
| `action` | string | yes | value |
| `value_class` | string | yes | value |
| `audit_ref` | string | yes | value |
| `ts` | string | yes | ISO-8601 timestamp |

**Example payload:**

```json
{
  "event": "high_value_action",
  "actor_id": "<actor_id>",
  "action": "<action>",
  "value_class": "<value_class>",
  "audit_ref": "<audit_ref>",
  "ts": "2026-05-27T12:34:56Z"
}
```

## Detection signal

```
value_class == 'high' without audit_ref OR audit channel silent
```

## Threats this catches

- Untracked high-value move
- Audit gap
- Bypassed scrutiny

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | High-value actions emit a value_class and an audit_ref, and silent audit channels alert. |
| 🟡 **Latent** | High-value classification exists but audit references are not co-emitted. |
| 🔵 **Collapsed** | Covered by T05 — economic action attribution already covers high-value cases. |
| ⚪ **Inactive (N/A)** | No high-value action class exists — all economic actions are uniformly low-value or the application has no economy. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
