# T01 — Value not duplicable

> Value must not be duplicable

## Why this matters

Conservation: every ledger transfer must increment a single monotonic sequence; sum-in must equal sum-out across the transfer set. Duplicate seq or imbalance is duplication.

## Telemetry

**Event:** `value_transfer`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `from` | string | yes | value |
| `to` | string | yes | value |
| `amount` | number | yes | numeric value |
| `asset_id` | string | yes | identifier |
| `ledger_seq` | integer | yes | numeric value |

**Example payload:**

```json
{
  "event": "value_transfer",
  "from": "<from>",
  "to": "<to>",
  "amount": 0.87,
  "asset_id": "<asset_id>",
  "ledger_seq": 42
}
```

## Detection signal

```
Duplicate ledger_seq OR conservation violated across transfer set
```

## Threats this catches

- Item duplication
- Currency dupe
- Exploit-driven inflation

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Every value transfer increments a monotonic ledger_seq, and conservation is checked across transfer sets. |
| 🟡 **Latent** | Transfers are logged but ledger_seq or conservation checks are not in place. |
| 🔵 **Collapsed** | Covered by T02 — balance snapshots subsume the conservation check. |
| ⚪ **Inactive (N/A)** | No transferable value — the application has no economy, currency, or assets that change hands. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
