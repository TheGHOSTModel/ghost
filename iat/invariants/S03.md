# S03 — Groups do not bypass safeguards

> Group behavior must not bypass safeguards

## Why this matters

Group decisions must reconcile to the sum of individual decisions. Divergence reveals group-bypass exploits where collective action escapes per-actor enforcement.

## Telemetry

**Event:** `group_action`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `group_id` | string | yes | identifier |
| `member_ids` | array | yes | list of identifiers |
| `action` | string | yes | value |
| `individual_decisions` | array | yes | list of identifiers |

**Example payload:**

```json
{
  "event": "group_action",
  "group_id": "<group_id>",
  "member_ids": [
    "<id_1>",
    "<id_2>"
  ],
  "action": "<action>",
  "individual_decisions": [
    "<id_1>",
    "<id_2>"
  ]
}
```

## Detection signal

```
Group decision differs from sum of individual decisions
```

## Threats this catches

- Group-bypass exploit
- Coordinated whitelisting
- Mob escalation

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Group-scoped actions emit individual decisions alongside the group decision, and divergence is alerted. |
| 🟡 **Latent** | Group actions are logged but individual member decisions are not co-emitted. |
| 🔵 **Collapsed** | Covered by S04 — coordination signals catch the same divergence pattern. |
| ⚪ **Inactive (N/A)** | No group-scoped actions — users act only as individuals. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
