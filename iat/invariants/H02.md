# H02 — Harm patterns detectable over time

> Harmful behavior patterns must be detectable over time

## Why this matters

Harm accumulates. A single signal may be noise; the same signature recurring across windows is a pattern. Time-bucketed scoring per actor is the substrate for repeat-offender detection.

## Telemetry

**Event:** `behavior_score_update`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `player_id` | string | yes | identifier |
| `window` | integer | yes | numeric value |
| `signal_type` | string | yes | value |
| `score` | number | yes | numeric value |
| `ts` | string | yes | ISO-8601 timestamp |

**Example payload:**

```json
{
  "event": "behavior_score_update",
  "player_id": "<player_id>",
  "window": 42,
  "signal_type": "<signal_type>",
  "score": 0.87,
  "ts": "2026-05-27T12:34:56Z"
}
```

## Detection signal

```
Cumulative score rising or repeated signals within window
```

## Threats this catches

- Repeat offender
- Slow-burn harassment
- Boundary-testing

## Status guidance

The status field above is **blank by default** — the framework is application-agnostic. Set it for your application using the criteria below.

| Status | When to mark it |
|--------|-----------------|
| 🟢 **Active** | Per-user behavior signals are aggregated into time-windowed scores, and rising or recurring patterns trigger review. |
| 🟡 **Latent** | Signals are emitted but not aggregated into per-user windowed scores. |
| 🔵 **Collapsed** | Covered by H05 (threat severity) — only high-severity classifications carry forward into per-user scoring. |
| ⚪ **Inactive (N/A)** | No repeated user-to-user interaction; or interactions are ephemeral with no notion of behavior history. |

When marking **Latent**, **Collapsed**, or **Inactive (N/A)**, also fill in the `rationale` field in the frontmatter with a one-line justification.
